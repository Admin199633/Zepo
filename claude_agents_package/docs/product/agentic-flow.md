# Agentic Flow for Claude Code - Mobile Poker Game

## Core Rule
For every feature:
1. Product Manager creates feature-spec.md
2. Poker Player Reviewer creates player-feedback.md
3. Senior UI/UX Designer creates ux-spec.md
4. Senior Mobile Game Developer creates technical-design.md
5. Senior QA Reviewer creates qa-test-plan.md
6. Implementation task breakdown is created
7. Only then implementation starts
8. After implementation, QA validation and signoff are required

## Gates
- Product Gate
- UX Gate
- Technical Gate
- Implementation Gate
- QA Gate
- Release Gate

## Required Feature Artifacts
- feature-spec.md
- player-feedback.md
- ux-spec.md
- technical-design.md
- qa-test-plan.md
- task-breakdown.md
- signoff.md

## Ownership
- PM owns scope
- UI/UX owns interaction hierarchy
- Developer owns architecture and implementation
- QA owns pass/fail and test coverage
- Poker Player owns realism and social feel
